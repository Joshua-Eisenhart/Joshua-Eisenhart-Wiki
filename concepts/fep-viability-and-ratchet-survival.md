# FEP, Viability, and Ratchet Survival

Viability is the safer survival spine. FEP is the support spine. Keep them related but separate.

## Viability frame

Viability theory studies dynamical systems under state constraints. The important wiki idea is not "organisms optimize a final purpose." It is:

```text
evolution must remain inside a constraint set
some initial states admit viable evolutions
others eventually violate constraints
```

This is closer to survival-under-constraints than broad FEP metaphysics.

## Crosswalk

| Viability object | Wiki use | Guardrail |
|---|---|---|
| constraint set | admissible region / survival boundary | must be explicitly named |
| viability kernel | states from which viable evolution exists | not assumed without dynamics |
| regulation map | selection of viable evolutions | not a teleological law |
| repellor / empty kernel | no viable continuation under given constraints | local to model |
| uncertainty | differential inclusion / contingency | not proof of active inference |

## Ratchet survival

The ratchet can use viability language when the local question is:

- which candidates remain inside the constraint set;
- which probes kill a candidate;
- what boundary the graveyard reveals;
- which re-entry is admissible after a failure.

That is not the same as saying the ratchet maximizes expected free energy.

## Blocked use

- Do not collapse viability into FEP.
- Do not use viability language to bypass local carrier/probe evidence.
- Do not turn survival into teleology.
- Do not claim a viability kernel without state space, transition relation, constraints, and negative cases.
