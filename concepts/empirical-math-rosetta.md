# Empirical Math Rosetta

## Purpose
This page processes the legacy `EMPIRICAL_MATH_ROSETTA.md` source into the current wiki.

Its useful kernel is not “math is merely metaphor.” The stronger claim is:

Mathematical objects should be read as emergent geometry of constraints acting on finite information structures, not as primitive Platonic forms.

This makes the page a support surface for the nominalist/constraint-first math spine.

## Current-use boundary
Strongest use:
- translation support for turning ordinary mathematical language into constraint/probe/support language.

Weak use:
- theorem status, sim evidence, or replacement for repo-current result surfaces.

Authority boundary:
- this is a legacy/source-derived Rosetta support page. It needs current sims/proofs before any specific mapped claim is promoted.

## Core translation table

| Ordinary math phrase | Constraint-first reading | Caution |
|---|---|---|
| number | distinguishability class or countable separation under a probe | do not assume primitive identity of units |
| set | admissible region / support class under constraints | do not treat membership as metaphysical essence |
| union | span / combined support that can hold alternatives | overlaps and compatibility must be checked |
| intersection | coexistence region / shared constraints | may be empty under noncommutation |
| function | rule mapping admissible distinctions to admissible distinctions | domain/codomain are earned supports |
| equality | indistinguishability under named probe/resolution | do not silently upgrade to primitive sameness |
| derivative | local response / constraint gradient | only meaningful on an admitted support |
| integral | path accumulation / flux over trajectory | path and measure must be earned |
| proof | admissibility certificate inside a formal system | formal primitives still need ontology audit |
| geometry | readout of constraint relations | not primitive background space |

## Fit to sim/math/geometry work
This Rosetta helps future sim/router pages ask the right questions:

- What is the finite support?
- What probe distinguishes the claimed objects?
- What constraint admits or kills the relation?
- What tool returns the witness?
- Is the geometry a primitive assumption, a sidecar projection, or an earned result?
- Does the result survive negative controls?

## Good MMM sentences
- Mathematics enters the system as constraint geometry, not as a primitive heaven of forms.
- A number is a stable distinction only after the probe can count it.
- Equality is an admitted shorthand, not a root object.
- A path integral is only honest after the path, measure, and support are earned.
- A proof certifies an encoding; the encoding still has to deserve its primitives.
- Geometry is what the surviving constraints let us read, not what we get for free.
