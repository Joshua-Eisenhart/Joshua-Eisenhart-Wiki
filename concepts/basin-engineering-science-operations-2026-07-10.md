# Basin Engineering As A Science Operation - 2026-07-10

Status: noncanonical vision, product, and research program derived from an
external advisory packet. This page is not evidence that one basin unifies the
listed domains.

## Core Opportunity

The broadest useful interpretation of the project is not merely a theory that
describes many fields. It is a candidate technology for working with finite,
history-dependent organizations under constraints:

| Operation | Executable question |
|---|---|
| map | What attractors, boundaries, viable states, bifurcations, and anomalies exist? |
| traverse | Which histories and ordered interventions reach each state? |
| preserve | Which constraints maintain viability, resilience, and recovery? |
| shape | Which constraint changes alter reachability or basin stability? |
| translate | Does a projection preserve dynamics, invariants, or predictions across views? |

These are five research operations, not five discovered engine stages,
personalities, ontological primitives, or a replacement for the 16-slot source
chart.

## Three Manifestations

### Foundational constraint theory

The foundational program asks which distinctions, compositions, histories,
and quotients survive finite probes before importing continuous geometry,
probability, causality, particles, or spacetime.

Its test is not whether established mathematics can be redescribed in Ratchet
language. It is whether the declared constraints force a risky structure,
exclude alternatives, or return several incomparable minimal survivors.

### Scientific discovery engine

The operational loop is already recognizable:

```text
generate candidates
-> quotient observational duplicates
-> apply ordered constraints
-> preserve failures and near misses
-> probe survivors and basin structure
-> run destructive controls
-> emit a bounded receipt
```

This can search explanatory structures rather than only numerical parameters.
The recent ECA observation-design packet demonstrates the honest negative
form: it searched, froze candidates, found controller defects, confirmed across
runtimes, and rejected all candidates without moving the validation gate.

### Governed intelligence architecture

The layers remain separate even when they cooperate:

```text
Codex Ratchet  earns or rejects scientific and mathematical claims
Holodeck       maintains bounded predictive state and reconstruction
Leviathan      executes governed actions, workflows, and receipts
Mesh           supplies independent witnesses, convergence, and adversaries
```

No layer may promote another by analogy. Lev runtime success is evidence about
execution; Ratchet receipts govern scientific admission.

## Cross-View Test Ladder

The cross-view ledger now uses six rungs:

```text
1 analogy
2 structural match
3 shared invariant
4 semiconjugate dynamics
5 cross-domain prediction
6 constraint necessity
```

The final rung is stronger than predictive transfer. It requires removing or
replacing a claimed root constraint and showing that the independently gated
result fails in both views. A shared implementation, shared labels, or one
domain ablation cannot establish root necessity.

The live proposal ledger has nine rows. Its highest current level remains
`structural_match`; `shared_attractor_admitted=false`.

## View Compiler

A potentially original long-term target is a view compiler:

```text
finite carrier
+ probe family
+ admissible transformations
+ coarse-graining
-> candidate descriptive languages
```

The output might recommend graph, automaton, algebra, geometry, stochastic
process, game, control system, or category language based on the quotient's
measured structure.

This remains far beyond the current estate. A valid compiler needs a benchmark
where competing mathematical languages make different predictions or permit
different compression, proof, or intervention performance. Merely naming a
familiar structure after seeing it would be ontology decoration.

## Decisive Near-Term AI Benchmark

The weak-model amplification experiment is already specified in
[[projects/leviathan-current/WIZARD_QIT_ENGINE_MAP_2026-07-02]]. It compares:

```text
weak-model-raw
weak-model-engine-harness
Fable-raw
```

under equal repo snapshot, task battery, tools, time, token, retry, commit, and
gate constraints. Eight task classes and deterministic metrics already exist,
including mutation survival, self-grading rejection, freshness compliance,
claim-ceiling precision, token cost, parity lift, and Fable gap closed.

The benchmark is not present in the current Lev checkout. The next action is
implementation and preregistration, not another design document. It must also
ablate terrain slots, parity, surprise boundaries, and EFE selection so extra
prompt structure or tokens cannot masquerade as engine capability.

## Basin Workbench Boundary

The advisory packet proposes one package with QIT, reaction-network, FEP, and
Lev adapters. The current decision remains narrower:

1. reuse the exact finite behavioral quotient as the first kernel;
2. run one source-native Attractors card;
3. earn one independently sourced semiconjugacy row;
4. implement the adaptive relation-directed observation policy;
5. extract a shared interface only after repeated working code appears.

A broad workbench built first would freeze domain assumptions into an API and
make shared code look like shared science.

## Product Wedge

The most defensible near-term value is governed model amplification and SME
workflow evidence, not foundational physics:

- modest-model improvement under equal budgets;
- inspectable multi-agent decisions;
- provenance and replay;
- stale-evidence rejection;
- correction without deleting history;
- company-specific language and object formation;
- bounded predictive event models.

This commercial route can be useful even if the deepest mathematical theory is
substantially revised.

## Claim Ceiling

Earned: a clearer five-operation research taxonomy, a six-rung cross-view
admission ladder, a specified but unimplemented model-amplification benchmark,
and a bounded implementation order.

Not earned: one universal basin, a cross-domain workbench, view compiler,
production Holodeck, general perception, solved alignment, consciousness,
biological prediction, four substages, the 64-stage schedule, Axis0, or new
physics.
