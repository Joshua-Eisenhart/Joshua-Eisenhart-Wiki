# Quaternion and Spinor Carrier Foundations

## Overview
This page exists because the Axis-0 / JK-fuzz / gravity genealogy repeatedly leans on `i,j,k` language, quaternion structure, and the `S^3` spinor carrier, but the public wiki did not yet expose that connection clearly.

The most stable current mathematical point is modest but important:
- `S^3` as the normalized spinor carrier is exactly the same object as unit quaternions / `SU(2)`
- this makes quaternion language part of the carrier geometry family, not just a decorative metaphor

## Core identification
The current-docs mapping already states:
- `S^3 spinor carrier = unit quaternions / SU(2)`

The fiber/spin reference pages make the same move more explicitly:
- identify `S^3` with unit quaternions
- use Hopf projection as a quaternion/spinor carrier structure

So quaternion structure belongs to the geometry/carrier layer of the project.

## Why it matters for Axis 0 genealogy
The owner-side genealogy linking `i`, `j`, `k`, `JK fuzz`, and gravity should not be read as if the quaternion language were imported from nowhere. The carrier geometry stack already contains a serious quaternion / spinor identification.

What remains open is not whether quaternions matter at all, but:
- how far the `i,j,k` carrier language should be pushed into the Axis-0 clock / fuzz / gravity line
- which parts are carrier geometry
- which parts are operator or shell-history structure
- which parts are still owner-doctrine rather than current theorem

## Relation to Pauli and Clifford structure
The same family also ties into familiar operator language:
- Pauli matrices satisfy the `su(2)` commutation structure
- Clifford / spinor pages already note the quaternion connection
- `i sigma_k` can be used as the standard basis of `su(2)`

So quaternion structure forms part of the bridge between:
- carrier geometry (`S^3`, Hopf, spinors)
- operator algebra (`su(2)`, Pauli)
- later Axis-facing attempts to read `i,j,k` more fundamentally

## Honesty boundary
What is safe to say:
- the wiki should explicitly expose the `S^3 = unit quaternions = SU(2)` identification
- quaternion structure is already part of the mathematical carrier layer
- this is relevant background for the `i,j,k`-side owner genealogy

What is not yet safe to say:
- that every owner-side `i,j,k` claim is already formally derived from the current carrier geometry packet
- that quaternion language alone closes the Axis-0 gravity / `JK fuzz` / `i`-scalar story
