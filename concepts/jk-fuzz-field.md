## Operational mathematical forms
The current docs give several compatible mathematical readings rather than one fully closed final theorem.

### 1. Admissible refinement set
In the rough-Axis-0 packet, the cleanest formalization is:
- admissible refinement index set: `Ω_r = {ω : ω labels a state in A(r)}`
- refinement count: `N(r) = |Ω_r|`
- refinement entropy: `H_Ω(r) = -Σ_{ω∈Ω_r} p_r(ω) log p_r(ω)`

In this reading, `JK fuzz` is the shell-indexed multiplicity of admissible future completions allowed by the current boundary state and constraint set.

### 2. Path-history ensemble
The same packet also gives a history form:
- one-step admissible map: `Φ(ρ) = Σ_α K_α ρ K_α†`
- multi-step branch state: `ρ̃_{α_1,...,α_n} = K_{α_n} ... K_{α_1} ρ K_{α_1}† ... K_{α_n}†`
- branch weight: `P(α_1,...,α_n) = Tr(ρ̃_{α_1,...,α_n})`
- path entropy: `H_path = -Σ P log P`

This is the path-ensemble / Kraus-history version of `JK fuzz`: not one rule firing, but an entire weighted field of possible continuations.

### 3. Constraint-manifold replacement for deterministic update
`MODEL_CONTEXT copy.md` gives the strongest outsider-facing translation:
- the state is the entire ensemble, not one selected branch
- deterministic one-rule update is the degenerate `j=k=1` case
- the fuzz is the computation

That makes `JK fuzz` a replacement for crisp deterministic rewrite dynamics, not just a visual metaphor.

## What it is not
The Axis-0 doctrine bridge is explicit that this should not be flattened into ordinary backward causation.

So `JK fuzz` should be read as admissible-future multiplicity and path-history weighting on a constrained support, not as a simple claim of classical retrocausality.
