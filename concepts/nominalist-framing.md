# Nominalist Framing

## Overview
This page records one current support framing for the wiki: a nominalist reading mode that treats abstractions as names, tools, and constraints rather than Platonic entities.

## Main points
- Use concrete terms, operators, carriers, and constraints first.
- Treat categories as useful labels, not reified essences.
- Prefer current-docs-aligned terminology when reconciling mixed or legacy pages.
- A longer-term goal is a wiki that helps readers stay closer to the user's nominalist, non-Cartesian, anti-Platonic style without turning this page into a front-door authority surface.
- If retrocausal language appears in nearby pages, treat it as a later candidate/support framing rather than as a primitive causal ontology.
