# Constraint Manifold Architecture

## Boot Rule

First load [[projects/codex-ratchet/current-decisive-frame-loader-2026-06-06]]. This page is the construction contract behind that first-screen frame.

The missing central object is the actual geometric constraint manifold.

Do not start this project from Xi, Axis0, FEP, Verlinde, Rosetta, gravity, physics, Jung labels, I-Ching rows, or QIT-engine language. Those are downstream candidates, readouts, controls, or source-pressure surfaces.

The object to make first is:

```text
M(C) = { x : x is admissible under the active constraint set C }
```

where the initial active constraint set is:

```text
C = {
  F01 finitude,
  N01 noncommutation,
  admissible probe rules,
  admissible composition rules
}
```

`M(C)` is the finite admissibility space. It is not automatically a smooth manifold, a spacetime manifold, a carrier geometry, a QIT engine, or a physics bridge.

## Decisive Separation

The constraint manifold and carrier geometry are different objects.

| Object | Role | Claim ceiling here |
|---|---|---|
| `C` | Active constraints and admissibility rules | Root charter for the construction |
| `M(C)` | Finite space of admissible configurations under `C` | Central object to make |
| Carrier/support geometry | A realization, readout, or support structure inside or mapping into `M(C)` | Candidate until earned |
| Axis map | A readout/function `A_i : M(C) -> V_i` | Coordinate/readout family, not primitive truth |
| Engine/bridge/scout | Later carrier, coupling, or readout candidate | Blocked until its inputs exist |

Carrier geometry can be proposed only after the finite admissibility object exists. A spinor, Hopf, Weyl, quaternion, tensor, graph, channel, or density-state construction may become a useful realization or readout, but the name of the carrier does not make the manifold.

## Required Build Order

The frame-loader order is:

```text
root constraints
-> M(C)
-> geometry on M(C)
-> axes as A_i : M(C) -> V_i
-> carrier/readout/coupling/engine/bridge candidates
```

This order is an embargo rule. A later surface cannot promote itself by using a familiar label.

Blocked until the manifold object exists:

- Xi / Phi0 bridge claims
- Axis0 closure
- gravity or physics claims
- FEP / active-inference bridge claims
- Verlinde / entropic-gravity claims
- QIT-engine admission
- Rosetta identity claims
- Fisher/Bures or information-geometry promotion
- terrain, flux, or carrier-geometry finality

Prior Rosetta, Xi, Bures, Fisher, FEP, and bridge-facing research results are useful as source pressure, controls, graveyard material, or follow-on tests. They are not the current frontier object.

## IGT Correction

In the owner pattern lane, `IGT` means Irrational Game Theory.

The core IGT pattern is the two-engine WIN/LOSE plus win/lose structure:

```text
outer / major loop: WIN or LOSE
inner / minor loop: win or lose
four terrain quadrants: WinLose, LoseWin, LoseLose, WinWin
two engine placements: Type 1 and Type 2 pattern placements
```

QIT operators, terrains, axes, Jung labels, I-Ching/taijitu tables, row tokens, and 64-address scaffolds are implementation, readout, grammar, or test surfaces. They do not replace IGT itself.

Do not acronym-collapse IGT into:

- infinite game theory
- information geometry theory
- information geometry topology
- generic game theory
- a QIT operator table

Irrational Game Theory can guide pattern search, row grammar, and engine placement hypotheses. It is not a proof surface until the relevant finite observables, controls, and maps are present.

## Language Discipline

Use status-bearing language:

- observed
- inferred
- proposed
- open
- blocked
- killed
- graveyard/control

Do not use bare identity or closure language:

- no "proves"
- no "is solved"
- no "the geometry is X"
- no "Axis0 is Y"
- no "Xi is the bridge"
- no "gravity/physics is admitted"
- no "QIT engine exists" from a scaffold
- no current manifold admission claim

The safest sentence form is:

```text
Observed source/result X supports proposed role Y under ceiling Z; it does not admit downstream claim W.
```
