# Constraint manifold layers run simultaneously

Worked out 2026-07-01..04 in Desktop-clone sessions; salvaged from Claude memory 2026-07-04. Source: `feedback_constraint_manifold_simultaneous.md`.

## Bottom line

The geometric constraint manifold is not a ladder you climb one rung at a time. All layers run at once. The engine operates at the most constrained level, with every layer active simultaneously — not at "the top rung" with lower rungs switched off.

## The structure

- Layer N (least constrained): the broadest allowed geometry.
- Layer N+1: more constrained, sits on top of layer N.
- Layer N+2: even more constrained, runs on N+1 and N at the same time.
- Engine level (most constrained): all layers active together, highest constraint.

Each higher constraint is more restrictive but does not replace the lower ones. It adds to them.

## Owner's correction that produced this

The owner corrected a prior framing that treated nested Hopf tori as a self-contained stage. Nested Hopf tori is "just one stage." Weyl spinors are more constrained, and run on all other layers at the same time. The engine runs at the most constrained level, where everything is active at once.

## How to apply

- Do not simulate layers sequentially as if lower layers turn off when higher ones activate.
- Every sim at layer N should have layers 0 through N-1 all running simultaneously.
- The engine is not "at layer 19" — it is at the level where all 19 layers are simultaneously active.
- The constraint manifold is nested simultaneous shells, not a sequential pipeline.

## Practical warning

Never prescribe "walk L4 to L11 in order" from the documentation ladder. The actual ratchet order is discovered empirically, by which admissibility tests bind — not read off the doc's inventory order.

Build plans that say "layer N is done, move to N+1" misread this structure. Doc order is inventory. Ratchet order is empirical.
