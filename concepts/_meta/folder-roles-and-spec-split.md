# Concept Folder Roles And Spec Split

`/concepts` is not required to track the repo perfectly.

Its strongest role is to preserve and cross-link:

- owner voice and source kernels;
- legacy genealogy;
- divergent conceptual options;
- candidate synthesis;
- reference math and research support;
- nominalist translation guidance;
- MMM and LLM salience material;
- Holodeck, Leviathan, TOE, personality, biology, consciousness, and technology reservoirs.

Repo status and executable truth belong elsewhere.

## Repo/Spec Mirrors

Pages that mirror live repo status should move or mirror into:

`/Users/joshuaeisenhart/wiki/specs/codex-ratchet/`

This includes:

- formal-scout readiness counts;
- sim estate counts;
- provider receipt status;
- tool/function/lego ledgers;
- contract mirrors;
- validator status;
- migration/status snapshots;
- dated process dashboards.

## Concept Page Repair Rule

Do not flatten concepts into repo status pages.

When a concept page has an issue, prefer one of these repairs:

1. label the layer: owner voice, generated source, legacy genealogy, wiki synthesis, support analogy, repo receipt, or current status;
2. fix broken source paths;
3. move live counts to specs and link to them;
4. soften only the words that claim proof or implementation;
5. preserve divergent models and source language.

## Nominalist Language Rule

Nominalist writing is a translation discipline, not a purge rule.

Human source language may use objects, subjects, causes, intentions, stories, or identity words. That does not make it invalid. Repair by adding the underlying constraint/probe/correlation reading, not by deleting the source voice.

Preferred stance:

- source voice is preserved;
- nominalist translation is supplied when needed;
- repo proof is earned only by executable receipt boundaries.

## Live Count Rule

Concept pages should not hand-maintain live counts.

If a concept needs status context, it should link to a specs/status page. If a count must appear in a concept page, it must be visibly dated and treated as an example, not current truth.
