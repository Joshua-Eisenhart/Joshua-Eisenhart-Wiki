# I-Scalar and Axis 0 Genealogy

## Overview
This page captures a genealogy the wiki had not mapped clearly enough: `Axis 0` did not arise in the repo as only a cut-entropy metric discussion. The owner-side and Axis-0-side docs repeatedly connect Axis 0 to:
- shell / boundary bookkeeping
- signed entropy and shell-gradient structure
- `JK fuzz` as admissible future multiplicity
- the `i`-scalar as a universal-clock candidate
- gravity / dark-energy / time correlations in the owner doctrine line

This page is not a promotion of those claims to solved current theorem status. It is a routing page so the genealogy is no longer lost.

## The i-scalar in the current docs
The current Axis-0 docs treat the `i`-scalar as a named but still-open family.

Strongest current statements include:
- the `i`-scalar should be the local reading of the universal entropy clock
- it should be frame-independent
- it should be derivable from RC-1 / finitude plus RC-2 / noncommutation
- its exact final derivation is still open

The rough-Axis-0 packet places `i`-scalar` inside the shell-bookkeeping / entropy-field family rather than treating it as a standalone later add-on.

## What Axis 0 is doing in this genealogy
The rough-Axis-0 packet says Axis 0 is doing several jobs at once:
1. base material / doctrine claim
2. shell and boundary bookkeeping
3. admissible future multiplicity (`j/k` fuzz)
4. entropy field and entropy gradient on that support

That means the owner genealogy is wider than the later executable cut-kernel packet.

## Strong current mathematical basin
Even with the larger genealogy preserved, the current docs still impose honesty fences.

The strongest mathematical basin currently documented is:
- admissible manifold `M(C)`
- shell / boundary decomposition
- bridge family `Ξ` from geometry or history into cut states
- entropy objects on those cut states
- signed entropy / coherent-information candidate family for `Axis 0`

So the wiki needs both truths at once:
- Axis 0 has a deeper owner genealogy involving `i`-scalar`, `JK fuzz`, and gravity work
- the current closed/earned mathematical packet is narrower and still open at bridge/cut unification

## Gravity genealogy
Several owner-facing surfaces connect:
- gravity
- dark energy
- time
- entropy
- entanglement
- `JK fuzz`
- `i`-scalar

The important wiki discipline is not to erase that genealogy, but also not to misreport it as already repo-closed. The doctrine bridge is especially clear here:
- gravity/time/dark-energy identifications remain proposal-facing unless formally derived
- shell/history and `Ξ`-bridge unification remain open

So the right public summary is:
- gravity is part of the Axis-0 genealogical motivation

- the full gravity-clock derivation remains open

Before this update, the wiki could make Axis 0 look like only:
