# Legacy Speculative Frameworks

## Overview
Older theory-heavy or highly synthetic frameworks. Useful as source history, but they are not the first-pass authority layer.

Use this page when the question is which large legacy framework a concept belongs to, not when the question is which current page should control interpretation.

## Internal split worth keeping explicit
Within this cluster, the current wiki now carries several different legacy branches:
- [[leviathan-framework]] -> governance / social OS / recursive methodology
- [[the-dark-empress]] -> civilizational build / collapse / cybernetics / cosmology mixture
- [[grandmaster-of-the-universe]] -> uncertainty-first foundational synthesis
- [[emotional-evolution-personality-system]] -> candidate engine grammar / personality-state package
- [[holodeck-docs]] and [[projective-holodeck-memory-model]] -> memory/world-model lineage

Those branches overlap heavily, but they are not interchangeable and should not all be summarized as one undifferentiated "legacy theory" surface.

## Role in the live wiki cluster
- **Strongest use:** branch-separation page for deciding which large legacy framework family a claim belongs to before reading any one stream too literally.
- **Weak use:** not the page for provenance strata, not the detailed comparison between the two major books, and not a current-synthesis page.
- **Authority boundary:** legacy branch-map surface only. It groups overlapping frameworks, but it does not decide current doctrine or runtime truth.

## Recommended reading order
1. Start with [[current-docs-vs-legacy-framing]] if the current-versus-legacy boundary is still unclear.
2. Use [[legacy-source-history]] for source strata and provenance.
3. Use this page to decide which major legacy branch actually carries the claim.
4. If the question narrows to the two major books, continue to [[dark-empress-vs-grandmaster]].
5. If the branch is collapsing into one fused legacy-theory blob, read [[legacy-speculative-theory]] as the caution page before extracting anything.
6. Then move to the specific branch page named here.
