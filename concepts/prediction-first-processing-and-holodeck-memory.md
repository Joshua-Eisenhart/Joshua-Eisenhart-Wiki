# Prediction-First Processing and Holodeck Memory

This page routes predictive-processing and active-inference support into the holodeck/world-model cluster without promoting it into consciousness or implementation proof.

## Core move

Prediction-first processing inverts the passive-input story:

```text
model predicts
sensory/readout error corrects
belief/world-model state updates
next prediction is emitted
```

For this wiki, the useful connection is to reconstructive memory and internal world-models. See [[holodeck-as-recall-space]], [[prediction-first-memory-vs-llm-memory]], and [[holodeck-qit-fep-leviathan-integration]].

## Crosswalk

| FEP/predictive-processing object | Holodeck/wiki neighbor | Guardrail |
|---|---|---|
| generative model | internal world-model / recall space | not exact replay |
| prediction error | correction signal | not direct truth |
| belief update | world-model revision | must preserve claim ceiling |
| future belief | counterfactual holodeck state | not proof of action policy |
| model evidence | persistence/admissibility neighbor | not ratchet admission |

## Re-entry loop

The safe synthesis from the attachment:

```text
prediction enters as candidate
constraint pressure kills or preserves
graveyard maps boundary
re-entry forms next belief/candidate cycle
```

That is a wiki routing analogy, not a claim that FEP derives the ratchet.

## Blocked use

- Do not write `prediction-first = consciousness proof`.
- Do not say the holodeck is implemented by Leviathan unless current repo evidence shows it.
- Do not claim QIT engine proof from predictive-processing language.
- Do not turn a world-model analogy into a sim result.
