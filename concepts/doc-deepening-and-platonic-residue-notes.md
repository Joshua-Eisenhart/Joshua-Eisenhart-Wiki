# Doc Deepening And Platonic Residue Notes

## Overview
Notes on deepening existing docs and cleaning Platonic residue. Connects to [[audit-platonic-residue-and-gaps]] and the nominalist reframing project.

## Core Issue
LLM-generated docs tend to use "X IS Y" language (Platonic identity) and "X creates Y" (causal language). The system's actual framing is: identity is not primitive (a=a iff a~b), and causality is not primitive (only correlation and co-variation under probes).

## What "Deepening" Means
Not adding more words. Adding:
- The derivation from first principles (constraint on distinguishability)
- Specific falsifiable predictions
- Negative controls and kill criteria
- Explicit mismatch notes when analogies break down

## What "Cleaning Platonic Residue" Means
Replacing:
- "X IS Y" -> "X and Y have not been distinguished under any applied probe"
- "X creates Y" -> "X and Y are statistically coupled; no probe has killed their correlation"
- "X causes Y" -> "X and Y co-vary; the arrow is notational, not ontological"

This is documented in detail in [[audit-platonic-residue-and-gaps]] and [[nominalism-in-this-system]].

## Priority Docs for Deepening
1. Thesis/cosmology doc (most Platonic residue)
2. Constraint surface and process doc (some causal language)
3. Reference docs (should use exact formal terms, not system ontology)
4. Mapping docs (should be explicit about fit/mismatch)

## What Deepening Does NOT Mean
- Adding more speculative theory
- Making docs longer without adding structure
- Claiming closure where the science is open
- Flattening nuance to make results look clean
