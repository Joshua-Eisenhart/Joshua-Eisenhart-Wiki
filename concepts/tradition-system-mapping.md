# Tradition -> System Mapping

## Overview
Date: 2026-04-05. Maps each external tradition to system claims it can support, and where it should not be overused. Rule: support tradition is not root ontology. Superseded by [[tradition-system-mapping-detailed]] for the detailed version, but preserves the high-level mapping logic.

## High-Level Mapping

### Formal Methods
Best support for: admissibility gates, witnesses/certificates, fail-closed checking, rejected candidate handling, invariant discipline. Do not use as: ontology of the universe, substitute for constraint substrate.

### Differential Geometry / Bundles
Best support for: geometry buildup, transport/connection language, carrier layers, spin structure, Hopf-fibration layering. Do not use as: proof that the system's ontology is "just geometry."

### Quantum Information / Quantum Shannon
Best support for: cut families, bridge/channel language, density operators, entropy/coherent-information language, witness language for entanglement. Do not use as: blanket metaphor for all correlation.

### Information Geometry
Best support for: distinguishability, statistical manifolds, metric structure on admissible state families. Do not use as: replacement for core constraint grammar.

### Active Inference / FEP
Best support for: candidate/witness update logic, prediction-first framing. Do not use as: root ontology (system doesn't require Bayesian probability as primitive).

### Evolutionary Epistemology
Best support for: variation-selection-retention framing, falsifiability pressure, graveyard-as-primary-output. Do not use as: proof that the system is literally biological.
