# Docs Framing Map

## Overview
This page separates the wiki into newer/current-docs-aligned material versus older/legacy material that is more inconsistent, speculative, or framing-dependent. It is part of the larger LLM constraint-harness project.

The current-docs side now has a supporting research overlay layer as well: the current spine, the evidence-backed inventory, and the external support pages that explain why the current stack is shaped the way it is.

Use provenance first, then framing:
- newer/current-docs-aligned = closer to the active wiki spine and preferred current-docs-aligned pages
- older/legacy/inconsistent = useful as source history, but not equally trustworthy as current framing

Nothing here is canon by default. This page is a routing/framing map, not an authority stamp.

## How to use this map
- Prefer current docs framing when two pages disagree.
- Treat legacy pages as provenance, not automatic authority.
- Expect some legacy docs to be internally inconsistent or to mix speculative, interpretive, and canonical tones.
- When a legacy page conflicts with current docs, keep both, but read the current docs first.
- When the question is about deeper formal grounding, read the current research overlays before reaching for legacy theory pages.
- When the question is about how the wiki should shape LLM saliency, read the harness scaffold and the support bibliography together.
- When the question is about autonomous wiki work, keep Hermes's built-in wiki-capable tools as the primary controller surface and treat outside automation layers as subordinate helpers only when a bounded tranche requires them.
- The future target is a nominalist framing where abstractions stay attached to concrete terms and constraints.
