## The rule (the whole contract in one statement)

Every layer, every nesting relation, and the full nested integration must climb the qubit
ladder one rung at a time; each rung must test the admissible entropy families for that
layer; nesting constrains which entropy/readout survives; ring-checkerboard and CA/QCA
surfaces are candidate finite runners, not decorative metaphors.

## The operational shape

For each LAYER (of the 27 in the GCM stack):
  - run the qubit ladder 1Q -> 2Q -> 3Q -> ... (1Q = exact Hopf/spinor; 2Q = boundary;
    3Q = the QIT floor (Cl(6), C^8); 4Q+ = support/stress and possibly NECESSARY for
    terrain/flux/runtime claims; 1Q/2Q never "proves the layer")
  - run the entropy-family sweep for that layer (candidates: Shannon, von Neumann, Rényi,
    Tsallis, min/max, conditional, mutual information, coherent information, entanglement
    entropy, negativity/log-negativity, shell-weighted, bridge-weighted, history-window,
    transport-weighted, operator-ordered; entropies are probe/readout functionals over
    explicit finite objects, never primitive causes)
  - run ring-checkerboard / CA-QCA surface variants where the layer admits a finite runner

For each NESTING relation (same gcm_object_id, same survivor/class/region lineage):
  - repeat the ladder on the nested object
  - repeat the entropy sweep UNDER the nesting constraints — the lower layers decide which
    entropy forms are available (the owner rule: each layer processes its own entropy
    forms, and the nesting constrains that)
  - test what survives quotienting, restriction, ratcheting — promote only the measure
    that survives the layer's geometry AND the negative controls

For the FULL INTEGRATION:
  - run the whole nested stack across qubit depth
  - full rich three engines (Julia canon + JAX batched + PyTorch graph/autograd),
    load-bearing not parity-decorative
  - include network/surface DYNAMICS (not just tables), the ring-checkerboard runner, the
    CA/QCA runner, and varied entropy families
  - controls where nesting, qubit depth, CA locality, ring topology, or the entropy choice
    is REMOVED

## The surfaces

The ring checkerboard supplies the finite support in three presentations whose equivalence
is checked, never assumed: flat nested checkerboard (local finite chart) = spherical
checkerboard (curved shell embedding) = nested rings/tori (fiber-loop / Hopf-torus
decomposition). The CA/QCA reading adds locality+homogeneity (the Margolus/brickwork normal
form; the 1D GNVW index as the computable chirality/flux invariant). Both are candidate
RUN-SURFACES for the carved manifold M(C,t) — admitted as runners only when realized on the
frozen substrate IDs, per the repo's substrate-first enforcement.
