## Pre-Axis Admission Pipeline

constraints → QIT-grounded machinery → negative-tested formal objects → Axis use

### 6-Phase Formal Process

| Phase | Action | Formalized | Simmed | Attacked |
|---|---|---|---|---|
| A | define | constrained object language | baseline live packet | non-admitted substitutes |
| B | ground | QIT carrier realization | carrier behavior | classical analog substitutes |
| C | refine | geometry/Weyl/transport | stagewise behavior | chirality/loop ablations |
| D | classify | candidate laws vs diagnostics | placement-sensitive runs | overpromotion attempts |
| E | embargo | keep unadmitted math out | stack checks | Axis leakage |
| F | elevate | admitted machinery → Axis | bridge/entropy/gradient sims | Axis-level negatives |

### 6 Admission Checks

| Check | Requirement | Fail mode if skipped |
|---|---|---|
| Constraint alignment | admissible under root constraints | ontology smuggling |
| QIT alignment | defined on finite carrier/density/operators | classical math leakage |
| Same-carrier honesty | carrier fixed when claiming necessity | fake necessity |
| Negative support | survives targeted falsification | arbitrary promotion |
| Placement clarity | classified before use | diagnostics promoted as laws |
| Axis embargo | can't be Axis math before passing above | axes on unstable substrate |

### Axis Embargo Rule

No math may function as an Axis object until it has been admitted as
constrained, QIT-grounded, simulated, and negatively tested machinery.
