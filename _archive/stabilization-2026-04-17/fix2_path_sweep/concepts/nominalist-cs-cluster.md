## Purpose

This page is the explicit cluster map/router for the nominalist-CS translation layer.

Use it when the question is not just "what is one page about?" but:
- how the nominalist framing fits the controller system

## Operational rules preserved by this cluster

### 1. Constraint/probe/exclusion before identity
A page in this cluster should not start by asserting what a thing is in the abstract. It should make the admissibility path visible.

### 2. Status labels are process labels, not confidence words
This cluster inherits the controller contract labels:
- `exists`
- `runs`
- `passes local rerun`
- `canonical by process`

### 3. Geometry-before-axis is preserved, not translated away
The CS framing must not flatten the scientific ordering. Queue semantics still keep geometry/chirality/operator packets ahead of bridge or axis promotion.

### 4. Evidence artifacts outrank summary prose
The cluster is only honest if summary pages remain subordinate to result files, truth-audit rows, lego ledgers, and current authority docs.

### 5. Metaphor handles are allowed but never load-bearing
JP/self-similar language may remain in bridge pages and outward-facing translation, but it must stay downstream of the nominalist/controller contract.

## How this cluster connects to the geometry lane

This cluster is not a substitute for the geometry-manifold lane. It is the translation/control layer that helps LLMs and developers describe that lane without smuggling:
- ontology before admissibility
- bridge claims before local geometry
- entropy before distinguishability
- summary closure before process closure
