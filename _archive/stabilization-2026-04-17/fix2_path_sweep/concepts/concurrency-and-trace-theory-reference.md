# Concurrency and Trace Theory

Concurrency theory studies systems where multiple processes execute simultaneously and interact through shared events. Trace theory, developed by Mazurkiewicz (1987), provides an algebraic framework for reasoning about equivalence of concurrent computations. This is relevant to the system's N01 (noncommutation) constraint and the multi-shell coexistence program.

## Core Structures

### Labeled Transition Systems (LTS)

An LTS is a triple (S, A, →) where:
- S = set of states
- A = set of actions (labels)
- → ⊆ S × A × S = transition relation

A process is a path through the LTS: s0 --a1--> s1 --a2--> s2 ...

### Trace Equivalence

Two actions a, b are **independent** (written a ⊥ b) if they do not interfere — executing them in either order reaches the same state. The **independence relation** I ⊆ A × A is irreflexive and symmetric.

A **trace** over (A, I) is an equivalence class of sequences under the equivalence generated by swapping adjacent independent actions. Formally: u ~ v iff u can be transformed to v by a finite sequence of swaps of adjacent independent letters.

**Mazurkiewicz language**: L ⊆ A* is a trace language if it is closed under ~ (if u ∈ L and u ~ v, then v ∈ L).

### Bisimulation

Two states s, t are **bisimilar** (s ~_B t) if for every action a:
- If s --a--> s', then t --a--> t' with s' ~_B t'
- If t --a--> t', then s --a--> s' with s' ~_B t'

Bisimulation is stronger than trace equivalence: it preserves the branching structure of possible futures, not just linear sequences.

### Refinement

Process **refinement** replaces an abstract action with a more detailed subprocess. Refinement is safe if it preserves observational equivalence (bisimulation or trace equivalence depending on the variant).

## Relevance to This System

### Connection to N01 (Noncommutation)

The system's N01 constraint says observables do not commute in general. In trace-theoretic terms:
- Non-commuting actions are **dependent** (not in I)
- The trace equivalence collapses only when actions commute
- The constraint cascade excludes sequences that cannot be quotient-identified under admissible swaps

The independence relation I on the system's operators is: A ⊥ B iff [A, B] = 0. Trace equivalence over this independence relation captures exactly which operator orderings are probe-indistinguishable.

This is a better CS-native rendering than saying "order matters" in prose. The repo's root constraint `AB != BA` gives an explicit dependence graph. The concurrency question is then: which reorderings are legal congruences on the execution word, and which reorderings change the admitted object?

### Connection to Shell Coexistence

The multi-shell coexistence program (stages 2-4) asks: which shells can be active simultaneously? In LTS terms:
- Each shell is a process with its own action set
- Coexistence = interleaving of shell processes
- Shells are compatible iff their interleavings form a trace language closed under admissible independent swaps
- Shells are incompatible iff some interleavings produce different admitted outcomes depending on order

This maps the coupling program onto the refinement/interleaving framework of concurrency theory.

### Connection to Topology-Variant Reruns

The topology-variant reruns (stage 4) test whether coupling results depend on carrier geometry. In LTS terms:
- Different topologies = different transition systems on the same action alphabet
- Topology-stable = bisimilar or trace-equivalent across topology variants at the chosen observation level
- Topology-sensitive = not bisimilar because branching structure or enabled actions change

## Key Results

1. **Mazurkiewicz trace theorem**: Trace languages over (A, I) are exactly the languages recognized by asynchronous automata — distributed systems where each component sees only its own actions.
2. **Trace monoid growth**: The growth rate of the trace monoid A*/~ is given by the clique polynomial of the dependence graph (A, D) where D = (A×A) \\ I.
3. **Bisimulation is decidable**: For finite-state LTS, bisimulation is computable in O(|A|·|S|·log|S|).
4. **Refinement preserves bisimulation**: If P ~_B Q and ref(a) is a bisimulation-preserving refinement, then ref(P) ~_B ref(Q).
5. **Partial-order reduction**: If independent actions are quotiented correctly, model checking can avoid exploring every interleaving. This is directly relevant to shell-coupling searches where naive enumeration of orderings is combinatorially wasteful.
6. **Guarded transition systems**: Practical controllers often encode progress as state plus admissible promotion rules. This matches the repo's truth-label discipline and queue gating more closely than a free-form task narrative does.
