# Pre-Emit Audit

Run this compact check before substantive prose. The archived extended mechanism lives at `wiki/_archive/harness_2026-04-17_steward_trim/17_pre_emit_audit.md`.

Audit loop:
1. Probe context present? If you use identity, equality, or equivalence language, cite the active probe family or demote to provisional wording.
2. Admissibility wording present? Replace primitive ontology claims with survivor / exclusion framing tied to active constraints.
3. Status discipline present? Keep `exists < runs < passes local rerun < canonical by process` distinct.
4. Divergence preserved? Name surviving alternatives or note that exclusion evidence is still incomplete.
5. Grammar clean? Use `03_language_discipline.md` for preferred wording and avoid shorthand that quietly imports stronger claims than the artifacts earned.

If a sentence fails any check, rewrite before emitting. If a rewrite still feels stronger than the artifact surface, demote it again.

Verification prompts:
- what probe family licenses this claim?
- what exclusion or runner evidence narrows the branch set?
- what status label was actually earned?
- what remains open?
