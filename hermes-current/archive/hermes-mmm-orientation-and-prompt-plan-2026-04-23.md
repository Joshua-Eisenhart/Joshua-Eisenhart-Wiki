## What this means for better prompts
Better prompts should be built in layers:
1. Pre-rule saliency preload for the main model: wiki harness/current spine plus L0 language-body alignment before task-specific prompting.
2. Per-lane/per-voice MMM index: a controller dictionary maps each visible route to its dense L0 + lane/voice language body.
3. Subagent boot: every visible lane/voice that is spawned runs as its own subagent and receives its saliency language before ordinary rules or task cards.
4. Voice mini-MMM: dense per-method language body for the active voice/lane.
5. Structural contract: measurable method requirements, such as Hume's observed/inferred/absent split or Popper's target/falsifier/check/classify sequence.
6. Task card: bounded goal, scope, out-of-scope, artifacts, verifier.
7. Output contract: literal section scaffold and receipt truth.
8. Validator: deterministic checks tied only to observed regressions.

The point is not prettier prompts. The point is recursive saliency alignment: outputs spoken in the system's language become future context that keeps later completions closer to the same nominalist/probe-relative manifold.
