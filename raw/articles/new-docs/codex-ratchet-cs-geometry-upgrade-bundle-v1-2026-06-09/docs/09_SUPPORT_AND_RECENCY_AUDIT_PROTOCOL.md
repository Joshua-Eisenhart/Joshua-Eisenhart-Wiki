# Support and Recency Audit Protocol

For each candidate library:

1. confirm current docs/release or acceptable maintenance state;
2. install/import in the Codex sim env or mark blocked;
3. run one micro-probe: one claim, one positive, one negative, one boundary;
4. record install friction and failure modes;
5. assign lane: baseline, tool-lego fit, canonical candidate, quarantine, avoid;
6. add a result receipt path before any promotion pressure.

No package is promoted because it is fashionable or powerful.
