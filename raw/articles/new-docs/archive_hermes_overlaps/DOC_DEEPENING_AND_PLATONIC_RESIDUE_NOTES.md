## Common Platonic residue patterns

### 1) Bare identity statements
Examples:
- "X is Y"
- "A and B are the same thing"
- "the essence of..."

Suggested fix:
- rewrite in probe-relative or relation-relative language
- use indistinguishability / observational equivalence when the literature supports it
- otherwise state the exact operational condition under which the relation holds

### 2) Causal overstatement
Examples:
- "X creates Y"
- "X drives Y"
- "X explains Y" when only correlation is shown

Suggested fix:
- replace with coupling, correlation, dependence, monotonicity, or monotonic loss of distinction as appropriate
- only say "causes" when the claim is actually supported

### 3) Compound terms stitched from real terms
Examples:
- "viability kernel under distinguishability constraints"
- other custom hybrid labels that are not standard terms

Suggested fix:
- prefer the standard term directly
- if a compound is needed for internal working use, mark it provisional and keep it separate from reference vocabulary

### 4) Over-clean hierarchies
Examples:
- primitive -> derived -> consequence -> analogy
- tidy ladders that hide overlap and mess

Suggested fix:
- allow a web of related concepts
- use explicit fit / mismatch notes
- do not force one neat derivation chain when the formal traditions do not support it

### 5) Narrative smoothing
Examples:
- one summary that erases alternate narratives
- collapse of different threads into one tone

Suggested fix:
- preserve branches
- keep competing interpretations labeled
- use challenge passes and provenance

## Suggested grammar rules

- Prefer concrete particulars over abstract nouns.
- Use existing formal terms first.
- Keep support traditions separate from the user's system claims.
- Distinguish reference vocabulary from mapping vocabulary.
- Mark uncertain claims as uncertain.
- Do not let a polished sentence outrun the evidence.

---

## Suggested editing rules

When deepening a doc:
1. Add exact formal terms first.
2. Add fit/mismatch notes.
3. Add what the term does not explain.
4. Add a mapping section only after the reference section is stable.
5. Add provenance if multiple threads contribute.

This is meant to increase depth without introducing made-up words or overly smooth framing.
