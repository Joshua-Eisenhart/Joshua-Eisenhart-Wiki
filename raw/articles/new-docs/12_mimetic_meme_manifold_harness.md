# Mimetic Meme Manifold Harness

## Purpose
This doc names the language layer that makes the harness work.
It is about salience-shaping, not decorative wording.

A harness is not only code, constraints, or sims.
A harness also needs a dictionary of aligned language that makes the right concepts salient to the model at boot.

## Local term
The repo’s actual source text uses "mimetic meme manifold" as the preferred term.
Legacy alias: "mimetic manifold" may appear in salvage text; treat as the same object unless explicitly distinguished.

## What the repo says it is
The clearest source wording defines it as the macro arena in which many engine instances couple, imitate, synchronize, fork, and prune through language/meme dynamics (Girard mimesis + Dawkins memetics).
It is an engine-of-engines layer, not just a per-terminal prompt layer.

## What it does
At the repo level, the mimetic meme manifold:
- couples multiple engine instances
- shapes how instances imitate, synchronize, fork, and prune
- provides the macro-language arena for Axes 7–12
- supports the later "holodeck-science" / IGT layer once measurable invariants exist

## Why this matters
LLMs do not merely parse meaning.
They also follow salience gradients.
If the language field is wrong, the model defaults to generic classical smoothing.
If the language field is right, the model stays near the real structure of the system.

So the harness-language dictionary is not a cosmetic layer.
It is part of the control system.

## Where "Mimetic Meme Manifold" sits
This local term sits across three existing ideas:
1. mimetic: the system should imitate the right form of reasoning
2. meme: the language units are reproducible cultural/linguistic units
3. manifold: the terms and phrases form a structured space of nearby boot states

So the manifold is not just a word list.
It is a structured field of language that orients the model toward the right operations.

## What the manifold should contain
A practical harness dictionary should include:
- exact names of the primitives
- allowed synonyms
- disallowed smuggled terms
- boot-specific vocabulary
- project-specific vocabulary
- examples of good and bad phrasing
- fixed negative-control phrases that should trigger caution

## What it should NOT do
- It should not become a theology of words.
- It should not replace the underlying math or sims.
- It should not hide uncertainty.
- It should not collapse multiple boot roles into one voice.
- It should not turn into vague inspirational language.

## Relation to sovereignty / control
If a group or project is controlled by its language field, that is not mystical.
It is coordination power.
Whoever controls the dictionary controls what becomes salient, what gets ignored, and what the model treats as real.

In that sense, harness-language control is a real form of authority.
But for this repo, the important part is not domination.
It is alignment: ensuring the model sees the same primitives the system uses.

## Relation to Leviathan / constraint engineering
Constraint engineering and Leviathan-language control are tightly linked in this repo’s framing because both are about:
- shaping admissibility
- constraining what can be seen as relevant
- maintaining separation of boots and duties
- preventing contamination across roles
- making the right structure more likely to survive

So the language manifold is one of the control surfaces of the harness.

## How to build it
1. Start from the root constraints.
2. Extract the actual terms already used in the repo.
3. Separate reference vocabulary from operational vocabulary.
4. Write boot prompts that bias salience without inventing new ontology.
5. Test the boot prompts with actual runs.
6. Keep the dictionary versioned and auditable.
7. Update the dictionary when evidence changes.

## How to test it
A good test asks:
- Does the boot prompt make the model use the right primitives?
- Does it preserve the distinction between boot roles?
- Does it resist generic classical smoothing?
- Does it improve the quality of the sims or docs?
- Does it keep different narrative frames separate without contaminating them?

## Practical bottom line
Yes: the harness needs a dictionary.
Yes: that dictionary shapes salience.
Yes: that is a real engineering layer.
But the dictionary should be built from the system’s actual constraints, not from decorative language.
