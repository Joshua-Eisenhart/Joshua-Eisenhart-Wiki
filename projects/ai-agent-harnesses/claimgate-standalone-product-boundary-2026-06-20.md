## Decision

ClaimGate should be treated as a standalone product repo. Codex Ratchet,
Wizard, Hermes, and Leviathan are source mines for useful patterns, not
runtime dependencies and not product-facing authority surfaces.

The useful extracted architecture is:

```text
soft side: model/council fanout proposes, critiques, scores
hard wall: deterministic gate checks named oracle evidence
promotion: only deterministic gate can admit
loop exit: gate-clean tied to oracle, not consensus
```

## External research: oh-my-openagent v4.12.0

Source:

```text
https://github.com/code-yeongyu/oh-my-openagent/releases/tag/v4.12.0
```

The v4.12.0 release was published by GitHub Actions on 2026-06-20. Its main
relevance is a Codex team mode surface: durable named Codex threads under a
leader session, team state in `.omo/teams`, member ownership areas, shared
artifacts, optional worktree paths, status updates, and cleanup/disband
mechanics.

Useful design patterns to strip for parts:

- durable team registry on disk;
- explicit member ownership/focus areas;
- leader-owned controller script for status, artifacts, bindings, and cleanup;
- guardrails against single-member teams, duplicate member focus, spawn-agent
  substitutes, symlink writes into guides, symlink delete escapes, and
  unresolved state paths;
- frequent small worker status updates so a leader is not blind during
  parallel work;
- wake/liveness hardening around stale sessions, polling, terminal errors,
  background output, and component reloads;
- release sequencing that fails before publishing when source-state gates have
  not landed.

ClaimGate assessment:

oh-my-openagent v4.12.0 is useful as a durable multi-thread/team control-plane
reference. It is not better than ClaimGate at the ClaimGate core problem.
ClaimGate's differentiator is the hard wall: model teams propose and critique,
but only deterministic gate checks tied to named oracles can promote.

Product extraction target:

```text
team registry / liveness / cleanup patterns
  -> ClaimGate model-pool and council receipt hygiene

hard-wall deterministic admission
  -> remains ClaimGate-owned, not delegated to team consensus
```

## Product boundary rule

ClaimGate zip/repo checks should enforce:

```text
no local absolute paths in receipts
no source-world runtime references
no missing relative requires
no committed bulky generated run estates by default
live model pool required for valid overall run
provider identities flexible per run
soft model lanes advisory only
hard deterministic gate is the only promotion path
```

The right product claim is:

```text
standalone product repo with compact live-model receipts and deterministic
hard-wall verification
```

Not:

```text
full Wizard runtime proof
all source repos bundled
consensus-based admission
fixed-provider deterministic live model calls
```
