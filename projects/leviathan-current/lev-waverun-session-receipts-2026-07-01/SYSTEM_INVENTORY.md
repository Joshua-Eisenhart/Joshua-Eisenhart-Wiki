# Lev + ClaimGate Patch — Complete System Inventory

## The overall goal

Lev (Leviathan) is a universal agent runtime: a graph-native system where every unit of work enters as a bounded Tick (INGEST → OBSERVE → PROPOSE → GATE → APPLY → UPDATE → EMIT), every mutation is a proposal evaluated by deterministic gates, and every completed unit produces an immutable receipt. The ClaimGate/WaveRun patch gives Lev a disciplined, multi-agent "cognitive wavegraph" layer on top of that kernel: durable cognitive members (agents, councils, subcouncils, management sidecars) with explicit manifests, organized into waves containing councils that loop (verify, reject, requeue) until clean, with Codex Ratchet supplying deep research/axiom/constraint/attractor-basin digging underneath. The patch is explicitly not a second proof system: every council/gate output becomes a proposal-only graph patch that only host Lev can admit, by independently recomputing the verdict rather than trusting a source projection's self-report. "Done" looks like a sendable Lev patch packet that proves, with real tests and receipts: cognitive member manifests are loaded (not just declared), model lanes across multiple providers report individually-receipted outcomes, nested waves and councils are distinct and can loop, a forced failure produces a typed FailurePacket and requeue, gate-clean only happens after repair, and the whole result reaches host Lev as a proposal that Lev itself admits or blocks.

**Hard invariants:**
- `C1_finitude` — all resources finite, every operation bounded, no infinite loops, no unbounded growth.
- `C2_non_commutation` — order matters; append-only event logs, immutable receipts, no silent mutation, deterministic replay.
- `C3_nominalized_reality` (assumed) — only validated canonical YAML executes; no semantic smuggling.
- `C4_ratchet_convergence` (assumed) — state transitions converge toward attractor basins; rollback requires explicit compensation; append-only.
- `C5_locality` (assumed) — every step has explicit scope; no omniscience; no ambient authority.
- No LLM, council, swarm, or branch can promote canon. No source projection can self-assert `live_lev_consumed`, `properly_integrated`, or `release_admission_allowed`. Model agreement is a signal that can trigger stricter verification, not faster promotion.

**Intended pipeline:**

Skill/MMM source packs → cognitive member manifests (Gate 1/2, loaded+hashed not declared) → councils/subcouncils assembled into looping waves (Gate 5) → model-lane formation across providers, each advisory-only with a receipt (Gate 0A) → gate loop kernel: candidate → verify → reject → FailurePacket → requeue → rerun → stop on clean/budget/circuit-breaker (Gate 3) → Codex Ratchet axiom/constraint/attractor-basin digging feeds proposal-only candidates (Gate 4) → management sidecars run beside the work plane (Gate 6/7) → council/gate outputs compile into a proposal-only graph patch, written as a ClaimGate steering source projection (Gate 9) → host Lev's `claimgate-steering` consumer independently recomputes the verdict and returns `host_consumed` or `host_blocked` → admitted state joins Lev's append-only graph/event log, with memory and "next seed" as derived views, not independent authority.

## Where we are — one-paragraph honest summary

End-to-end, the chain that actually runs today is: a deterministic execution spec → ClaimGate gate loop → graph-patch proposal → host-side independent recomputation → `host_consumed`/`host_blocked` → real graph mutation, all exercised live in this session (`demo --json` → `admit-graph-patch` → `apply-plan` → `apply-graph-patch`, ending `graphMutationApplied: true`).
