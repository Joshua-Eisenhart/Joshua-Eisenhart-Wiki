OG architecture:

```text
context/world state
-> multiple generated predictions
-> projection into a virtual or physical world surface
-> sensed actual input
-> prediction-error correction
-> confirmed semantic hash / memory update
-> allostatic or action update
-> repeat
```

Components:

- Projection: LLM/CDO or predictive-coding engine generates forward models, sometimes in parallel.
- Error correction: camera/sensor/sim input is compared to prediction with KL, cosine similarity, pixel/MSE, trace distance, or expected-free-energy style objectives.
- Memory: confirmed matches become semantic hashes with context/intent metadata, vector storage, sparse distributed memory, and chain recall.
- World interface: first a Gym/Godot/libtcod world, later physical cameras/projectors/sensors with projector-camera calibration.
- Action/allostasis: future-state prediction selects actions that minimize expected future error/load.
- Rationalization: in the Humean branch, PyTorch perception runs fast and LLMs narrate/explain slower.
