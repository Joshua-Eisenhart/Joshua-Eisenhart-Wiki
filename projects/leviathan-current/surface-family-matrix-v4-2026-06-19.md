# Leviathan Surface Family Matrix

Leviathan is not one UI. It is a plural surface strategy.

## Surface families

- AgentPing / dashboard / human-loop;
- LevUI IR;
- GenUI;
- voice;
- vision/capture;
- browser ops;
- desktop service manager;
- lev-now;
- mobile/expo;
- CLI;
- MCP / HTTP / gRPC / WebSocket;
- prompt-stack sessions;
- SDLC/autodev flows.

## Matrix fields to track

For each surface, record:

- source path;
- package/app/plugin owner;
- target user;
- current proof rung;
- required dependencies;
- human-loop or agent-loop role;
- security/permission boundary;
- product status.

## Safe verdict

Leviathan has many real surface directions. A surface-family matrix is more honest than a single "the app works" claim.
