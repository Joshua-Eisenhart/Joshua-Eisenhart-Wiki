# One System, Three Perspectives - 2026-07-02

## Charter Claim

This document is the CR / Lev merger charter.

Owner thesis: **lev-live, the mesh project, and the CR merger are the SAME object**. They are one recursion:

```text
inventory -> exposure -> independent witness -> correction -> accumulate
```

The same recursion is instantiated three ways:

| Perspective | Concrete loop |
|---|---|
| Runtime | Lev: `capture -> refine -> judge -> execute -> graph` |
| Network protocol | Mesh: `expose -> peer-audit -> converge -> settle` |
| Evidence estate | Codex Ratchet: `sim -> receipt -> gate -> registry` |

This is an isomorphism of loop shape, not a collapse of authority. The runtime, network protocol, and evidence estate have different owners, repo topologies, trust regimes, and claim ceilings. Those differences are preserved below.

## Divergences Preserved

These are not rough edges to smooth. They are load-bearing distinctions.

### 1. Execution vs Normative Authority

Lev runtime can execute, judge admissibility, emit receipts, and update graph state. It cannot declare human, social, or research "quality" by execution alone. [Gate Doctrine](./GATE_DOCTRINE_ADMISSIBILITY_NOT_QUALITY.md) is explicit: code says what survived declared checks; it does not say "this is good." [Leviathan Construct To Mesh Map](./LEVIATHAN_CONSTRUCT_TO_MESH_MAP_2026-07-02.md) fences public scoring, currencies, family/social systems, events, and land-policy constructs. The merger charter must therefore keep technical admission separate from normative authority.

### 2. Repo Topologies

Lev live, CR evidence estate, and the wiki charter live in different topologies. [WF7 Lev Live Map](/private/tmp/claude-501/-Users-joshuaeisenhart-Desktop-Codex-Ratchet/be658e3e-11bb-4bcc-8728-d7ac9e255db1/scratchpad/wf7_lev_live_map.md) maps `origin/main` after 115 new commits, current `core/eval`, `plugins/sim-witness`, `core/poly`, `core/reconciler`, world-model, graph, and heartbeat surfaces. [WF7 CR Into Lev Decomposition Map](/private/tmp/claude-501/-Users-joshuaeisenhart-Desktop-Codex-Ratchet/be658e3e-11bb-4bcc-8728-d7ac9e255db1/scratchpad/wf7_decomposition_map.md) maps CR `system_v7` through `system_v4` evidence roots and says raw result estates, Julia carrier, guarded runner, CR registry, Wizard v4.3 route truth, and quarantine archives stay CR-native unless proven otherwise. [CR / Lev Merge Project Plan](./CR_LEV_MERGE_PROJECT_PLAN_2026-07-02.md) also has a patch-flow hold until JP's upstream rehoming paths and receipt anchors stabilize. Selective evidence transport is the merger. Repo copy is not.

### 3. Two-64s Owner-Reserved Decisions

The two-64s issue is not resolved by showing the runtime loop or mesh loop. [Ratchet State By Tier](../constraint-core/RATCHET_STATE_BY_TIER_2026-07-02.md) marks the L6 `TWO-64s` reading, `2x8x4=32/engine vs 16x4`, as `OWNER-RESERVED, open`, and says the canon and hypothetical lanes are never merged. This is the class of owner decisions where evidence can prepare the ground but cannot silently select the doctrine. The merger charter preserves that boundary.

### 4. Single-Node vs Network Trust Regimes

Host-local trust work is not peer identity. [WF7 Mesh v0 Gap Analysis](/private/tmp/claude-501/-Users-joshuaeisenhart-Desktop-Codex-Ratchet/be658e3e-11bb-4bcc-8728-d7ac9e255db1/scratchpad/wf7_mesh_gap.md) says the old branch's host-local HMAC work is portable as implementation technique and tests, but not sufficient as cross-node trust because it is symmetric and scoped to one OS user/host. Mesh v0 remains a two-node content-addressed seam, not node #3 readiness or autonomous federation. Network trust needs signed bundles, node identity, anti-replay, and receiver-side verification before transport claims expand.

## Charter Boundary

The merger is one object because every surface repeats the same attractor:

```text
inventory produces exposure;
exposure invites independent witness;
independent witness forces correction;
correction accumulates only through receipts;
the accumulated state becomes the next inventory.
```

The merger is not one object in the sense of one repo, one authority, one trust domain, one runtime, one social doctrine, or one registry. The point of the charter is to keep the attractor unified while preserving the divergences that make the loop honest.
