This map treats the owner-authored Leviathan v3.2 text as a human social framework, then maps each major construct to the avatar / agent-network realization that currently exists or is nearest buildable in Lev / Codex Ratchet / mesh practice. The governing ceiling is the same as [Mesh Node Protocol v0](./MESH_NODE_PROTOCOL_V0_2026-07-02.md) and [Gate Doctrine](./GATE_DOCTRINE_ADMISSIBILITY_NOT_QUALITY.md): receipts, patches, gates, witness packs, content addresses, and local reruns can prove only their own rung.

Stage meanings:

| Stage | Meaning |
|---|---|
| `exists` | A current artifact, protocol, receipt, gate, patch flow, or local-run evidence already instantiates the construct in the agent mesh. |
| `buildable-now` | The construct can be built as an avatar/agent-network feature using current surfaces, but the specific feature is not implemented as such. |
| `fenced-later` | The construct is social/economic/normative, privacy-sensitive, or product-scale; current artifacts may inspire it but do not implement it. |

## Fenced List

These constructs are normative-only or materially under-implemented today:

| Construct | Why fenced |
|---|---|
| Public attractiveness / social credit scoring | Current agent mesh should use receipt rungs and capability proofs, not personal score exposure. Human scoring needs major privacy, safety, and product review. |
| SBC / CRC / ECC as currencies | Mesh v0 explicitly stage-gates them as design candidates. They can be accounting columns before they are tokens. |
| Family/intergenerational social system | Human social institution, not an agent-network construct. Only mentorship patterns map safely today. |
| Anti-Internet social matching/events | Requires real-world safety and privacy design; no current implementation. |
| Progressive LVT | Economic/governance policy. No current Lev/CR implementation. |

## Notes

- The strongest current realization is not a human society product. It is a two-node artifact mesh: inventory, exposure, peer witness, correction, and service already operate through patches, witness packs, receipts, gates, and returned findings.
- The three Mirrors are buildable as avatar/agent mirrors today, but not as the full human scoring systems described in v3.2.
- The currencies should remain accounting dimensions until the system has stable service/resource/ecosystem ledgers and an explicit economic design review.
- The human social constructs should not be back-filled into agent claims by analogy. Use receipt rungs and gate doctrine for agent credentials; keep personal, family, dating, land, and social-score constructs fenced.
