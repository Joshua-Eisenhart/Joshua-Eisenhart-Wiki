## Spinor Network Priority

The canon being requested is not "my language as canon." It is:

```text
finite spinor networks
```

Use spinors because F01 and N01 pressure the model toward the simplest finite noncommutative carrier:

- F01 demands finite carrier/probe/operator/path sets.
- N01 demands order-sensitive or noncommuting composition.
- The minimal finite quantum carrier gives `C^2`, SU(2), Pauli probes, spinors, chirality, and density readouts.
- Hopf geometry gives `S^3 -> S^2`, separating spinor/fiber structure from Bloch/density/base readouts.
- Entanglement gives the first nontrivial multi-spinor support/readout structure.
- Three spinors / three qubits is the minimum currently flagged for nontrivial network/nonassociative tests.

Tensor-network language may be useful for implementation, contraction, or PEPS-style finite support. But future workers should not reframe the project as tensor-first. The intended object is a finite spinor network with density/probe readouts.

## Nonassociativity Changes The Manifold

If nonassociativity is added, the candidate manifold cannot be treated as ordinary point-set geometry with a globally associative composition law.

The object becomes closer to:

```text
finite spinor-network admissibility space
+ local order-sensitive channels
+ bracket-sensitive composition
+ holonomy/curvature-like readouts from path and bracketing differences
```

Consequences:

- A state is not only "which finite support point" but also "which admissible history/bracketing produced this readout."
- Geometry becomes local and connection-like: path order, bracketing, and transport matter.
- Two networks with the same reduced density readouts can still differ by admissible composition history.
- A two-spinor system can show entanglement/chirality, but a three-spinor system is the minimum where associator witnesses can exist.
- Axis/IGT/QIT readouts become coordinates over a history-sensitive spinor-network object, not labels pasted onto a static space.
