# Codex Ratchet Wiki Upgrade Pack Intake - 2026-06-19

## Intake Verdict

The Codex Ratchet report and ZIP are useful as a wiki upgrade pack. They are not repo authority and do not admit, promote, rerun, or canonicalize any sim/result.

Audit result: **pass with cautions** for wiki reduction; **not safe for A2 doctrine ingestion** without a separate memory-admission pass.

## Why Cautions Apply

- The pack is explicitly targeted, not exhaustive.
- The deep-research report was based on public GitHub/wiki visibility limits and should not outrank local repo/wiki evidence.
- The pack carries many generated draft pages; direct bulk import would inflate the wiki without proving current repo truth.
- Several pages mirror status and process surfaces. They must stay dated mirrors, not live status claims.
- The pack does not provide a fresh validator/result rerun and does not settle `M(C)`, QIT-engine, Axis0, bridge, physics, layer, or manifold claims.

## Accepted From The Pack

- Add a compact Codex Ratchet load-order card so agents are not forced to start inside the dense project front door.
- Add repo-authority and repo-to-wiki ingestion routers that keep exact repo reads above wiki prose.
- Add spec mirrors for repo layout, current docs map, process-contract refresh, status-label crosswalk, SIM_TEMPLATE result shape, sim-mode Full Wizard runbook shape, and generated snapshot boundaries.
- Add reusable templates under `templates/codex-ratchet/` for claim ceilings, router cards, result-evidence tables, sim-result intake, spec mirrors, project routers, and repo-to-wiki tranches.
- Keep the pack index as a map of what was reviewed, not as a command to import every file.

## Rejected Or Deferred

- Full ZIP import: deferred.
- Any generated page that would require a fresh repo result count before use: deferred to exact-read/rerun.
- Any claim that a scratch diagnostic is admitted/canonical: rejected.
- Any claim that the wiki can substitute for `AGENTS.md`, process docs, validators, or result receipts: rejected.
- A2/A1 memory routing: deferred until a separate guard run.
