# Unseen Finite Predictive Objects v1 - 2026-07-10

## Verdict

`RED_PROCESS_FAILURE_AFTER_SEAL`

This page mirrors a live Codex-Ratchet scratch-diagnostic result. It is not
wiki canon and does not promote object perception.

## What Was Real

- A fresh `ufpo-v1` finite registry excluded all v0 machine and bounded
  predictive-signature identities.
- The registry contained 2,367 eligible four-state processes and froze disjoint
  128/32/32 train/validation/test objects.
- Object identity was exact uniform-start binary-word prediction through length
  eight, not a learned label or machine hash.
- Sixteen full-observation-horizon-matched pairs partitioned the fixed test
  objects as a challenge set, without claiming universal hard negatives.
- Julia independently reconstructed exact rational semantics.
- JAX independently checked registry enumeration, split identity, exact
  numerators, view-PRNG structure, leakage constraints, and solver controls.
- PyTorch ran a real GRU/DeepSets train-validation preflight over 16 arm/seed
  cases without opening or generating test views.
- One tracked source seal was created only after the three runner contracts were
  reconciled and affected preflights rerun.

PyTorch is a broad tensor, autograd, graph, network, differentiable-dynamics,
optimization, and optional learning toolkit. This experiment used one scoped
trainable role; its failure does not define PyTorch's capabilities.

## What Failed

The sealed Julia exact-semantic leg passed in its required strict carrier
environment. The JAX sealed workhorse leg passed locally and stayed pending the
controller. A first Julia run in the wrong project failed only its runtime gate
and is retained separately.

The sealed PyTorch path opened the frozen test records, then its order-2
Laplace baseline sent one unbatched record (`ndim=2`) through a schema assertion
requiring batched tensors (`ndim=3`). The bug is independent of the number of
views per object. It raised
`RunnerError: visible tensors have inconsistent shapes` before producing
`pytorch_result.json`. The independent controller therefore emitted red for a
missing required learner result.

No thresholds were changed, no best seed was selected, and the source was not
patched and rerun after test opening.

## Meaning For Engine Claims

Earned: exact finite-object registry machinery, exact semantic reconstruction,
JAX exhaustive verification, clean source sealing, and a real falsifying
sealed-path failure.

Not earned: unseen learned object retrieval, perception, object creation, 16
unique intelligences, four dual-ratchet-emergent substages, 64 running engine
stages, MMM or ontology admission, Axis0 closure, FEP, physics, life, or
consciousness.

A pre-test `/tmp` validation run suggested weak same-object retrieval and little
separation from temporal shuffle, but that summary was not admitted as a
tracked packet receipt. It cannot substitute for test metrics that never
completed.

## Required v2

v1 cannot be prospectively repaired because its sealed one-shot evaluation was
consumed. The test machine tables were always plaintext in the preregistered
manifest; they were hidden from model input, not from repo readers. A valid
retry requires a new namespace, fresh test objects, new manifest, new seal, and
a synthetic pre-seal rehearsal of the exact sealed-test call graph covering
unbatched and batched baseline inputs without using v2 test records.
