# Current Repo Status Claim Ceilings — 2026-06-19

## Bottom line

Most wiki pages should not make live repo-state claims. They should route to exact repo surfaces.

## Claim ceiling table

| Page/source family | Safe claim | Unsafe upward inference |
|---|---|---|
| Wiki project front door | current routing surface | executable status |
| Spec mirror | repo-facing summary | repo authority |
| Formal scout index | generated readiness/index snapshot | canonical promotion |
| Sim estate index | generated estate/index snapshot | final manifold/axis/engine |
| Result JSON | exact result content | broad stage completion |
| Scratch diagnostic | useful pressure/control | admitted object |
| Source-processing page | provenance/router | owner doctrine or sim evidence |
| CocoIndex search result | semantic pointer | exact evidence |

## Required phrase

When copying status into a wiki page, include:

```text
Snapshot/source: <repo path>, generated or checked <timestamp>.
Claim ceiling: <routing/status snapshot/scratch only/etc.>.
Promotion: not implied.
Exact-read required before reuse.
```

## Wiki policy

Use fewer numbers in concept pages. Link to spec/status mirrors and exact repo artifacts instead.
