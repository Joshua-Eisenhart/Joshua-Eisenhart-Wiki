## Existing Kernel Pieces

The proposed `View` interface should not be installed as an abstract Python
class yet. Its operations already have earned finite representatives:

```text
probes       -> finite probe response functions
quotient     -> stable behavioral partition refinement
evolve       -> exact action-labelled quotient transitions
invariants   -> partition, SCC, cycle, basin and relation ledgers
ablations    -> false quotient, action swap, relabel, edge erasure and mutations
```

These live in
`system_v7/sims/finite_probe_behavioral_object_engine_v0`. The missing object
is not an interface. It is a second, independently sourced domain dynamics and
a measured residual for

```text
Q_d(T(x)) ~= T_d(Q_d(x)).
```

The proposal-only `cross_view_attractor_ledger_v1` already fails closed when a
row claims semiconjugacy without that receipt. A new package tree would only be
useful after two executable rows reveal stable repeated code.

## Attractor Tool Truth

A fresh isolated rerun used:

```text
Attractors 1.37.0
DynamicalSystemsBase 3.18.1
IntervalArithmetic 1.0.9
```

`AttractorsViaProximity` and `basins_of_attraction` recovered the expected two
attractors and basin assignment on a tiny bistable map. This is `api_smoke`, not
`claim_load_bearing`.

The official Attractors API supports basin fractions, basin entropy, parameter
continuation, tipping probabilities, and basin-boundary methods
([Attractors API](https://juliadynamics.github.io/DynamicalSystemsDocs.jl/attractors/stable/api/)).
ReachabilityAnalysis represents over-approximated reachable states as flowpipes
and reach sets
([official introduction](https://juliareach.github.io/ReachabilityAnalysis.jl/stable/tutorials/linear_methods/introduction/)).

The first claim-bearing Attractors card should use one current source-native
dynamics, not four unrelated domain analogies. It must compare native,
reversed, grouped, commuting, and one-axis schedules and report basin fractions,
boundary sensitivity, tipping parameters, and tool-removal demotion.

## Formal Tool Placement

### Maude

Maude is a good match for finite, bracket-explicit, nondeterministic histories.
Its rewriting-logic surface can search reachability and analyze executable
rewrite theories ([Maude manual](https://maude.cs.illinois.edu/manual.pdf)). It
is not installed on this host.

The first card should ask one bounded question:

```text
Can distinct local bracket/order histories remain noncommuting while reaching
one probe-equivalence normal form, and which rule mutation destroys that
macro-confluence?
```

It should not encode the whole foundational theory.

### Lean and Physlib

The suggested Lean-QuantumInfo route has moved. Its repository states that new
development moved into Physlib
([repository notice](https://github.com/Timeroot/Lean-QuantumInfo)). The live
Ratchet already built Physlib's associative quantum DPI module across 8,617
jobs with no source `sorry` or `admit` hits. New theorem cards should extend
that current surface and avoid reproving elementary facts already upstream.

### TLA+ and Apalache

Apalache symbolically checks typed TLA+ specifications by translating bounded
executions into SMT constraints
([official documentation](https://apalache-mc.org/docs/apalache/index.html)).
It is absent locally. Its correct first owner is Lev because receipt
immutability, quorum, idempotency, stale projection, replay, and append-only
graph behavior are distributed-protocol obligations. CR should consume a
content-addressed proof receipt rather than host a duplicate protocol model.

## Algebra and Biochemistry

OSCAR genuinely integrates GAP, Singular, polymake, Nemo, Hecke and
AbstractAlgebra through Julia
([OSCAR architecture](https://docs.oscar-system.org/stable/General/architecture/)).
That makes it relevant when an MSS census has algebraic isomorphism classes the
current finite exact code cannot classify. It is not a reason to rebuild the
existing finite survivor enumerators.

Catalyst remains a plausible reaction-network view, but no package or
claim-bearing receipt is present. Its first use must be narrower than
"biochemistry is the engine": compare schedule interventions on a frozen
autocatalytic network and test viability-basin persistence under order reversal,
grouping, commuting, and one-axis controls.
