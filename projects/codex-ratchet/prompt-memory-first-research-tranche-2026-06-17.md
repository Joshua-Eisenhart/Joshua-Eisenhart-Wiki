# Prompt Memory First Research Tranche — 2026-06-17

## Purpose

This page is the first research-backed tranche for processing prompt-like Claude/Codex/Hermes memory into wiki form.

It does three narrow jobs:

1. names candidate owner-intention kernels that should be tested against a provenance ledger;
2. attaches research comparators that can support, challenge, or discipline the language;
3. sets claim ceilings so memory rows do not become proof or canon by sounding polished.

It is downstream of [[projects/codex-ratchet/prompt-memory-intention-formalization-router-2026-06-17]].

## Research-status note

Live web search was unavailable in Hermes during creation (`FIRECRAWL_API_KEY` / `FIRECRAWL_API_URL` not configured). To avoid pretending, this tranche uses:

- direct arXiv API metadata checks for named papers;
- existing wiki research pages and exact local reads;
- bounded comparator use, not broad new literature synthesis.

Source-status fence: an arXiv ID in a comparator table means the paper handle and metadata were API-confirmed unless a row also cites a local exact-read wiki page or extracted paper text. It does not mean the full paper was read, digested, or promoted as source-backed content for this tranche. Treat arXiv-only rows as comparator handles for bounded follow-up, not as completed literature synthesis.

## Ledger status

No bounded raw-memory quote ledger is included in this tranche. Kernels A-D are candidate routing syntheses and research comparator queues only. They must not be cited as owner doctrine, admitted concept doctrine, or evidence of Josh’s final view until ledger rows establish the owner / assistant / source split.

## Source split rule

A memory row can enter wiki prose only after this split:

| Layer | Meaning | Allowed output |
|---|---|---|
| Owner kernel | A repeated owner prompt, correction, source fragment, or stable framing signal | source ledger / route patch / concept page note |
| Owner uncertainty | Owner marked ambiguity, fork, correction, or live alternative | open question / fork table |
| Assistant elaboration | Claude/Codex/Hermes generated explanation, plan, or synthesis | comparator only unless tied to owner source and evidence |
| Repo/wiki evidence | receipt, sim, validator, existing source page, or exact wiki anchor | evidence citation with status label |
| External comparator | paper/book/framework that helps test language | support or falsifier lane only |
| Claim ceiling | explicit max status | no promotion above checked evidence |
