## Stale-page triage

Use `wiki_drops/backlog/stale-page-checklist.md` before editing older pages. Main stale signals:

- Old status counts without generated timestamp.
- `canonical`, `verified`, `admitted`, or `complete` without layer and evidence path.
- Old concept mirror that has a newer spec mirror.
- Raw source memory used as owner doctrine without bounded quote/confirmation.
- Source-processing page that lacks `claim_ceiling`.
