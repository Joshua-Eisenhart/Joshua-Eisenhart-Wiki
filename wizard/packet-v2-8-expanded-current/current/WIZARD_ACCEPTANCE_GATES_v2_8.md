# Wizard Acceptance Gates — v2.8

v2.8 is the expanded recovery branch after v2.7 collapsed. These gates decide whether a packet has escaped collapse.

## Size Gates

- Main full MMM should remain reservoir-scale. Current restored target: v2.5 main full MMM scale, 37,959 lines.
- Full mini-MMMs should remain reservoir-scale. Current restored target: 24 full mini-MMM files, 43,978 lines.
- Full Wizard General should remain method-scale, not route-label scale. Current restored target: v2.5 full reference scale, 859 lines, plus v2.8 source-extraction addenda.

Size is a guardrail, not a proof. Padding with labels fails.

## Runner Gates

- Runner detects packet version from manifest.
- Runner prints `Wizard v2.8` for v2.8 packet.
- Runner does not require v2.7 alias files.
- Every visible route has a receipt.
- Full live Wizard waves are mixed-model waves. Every wave should include Sonnet-heavy breadth, sparse Opus correction, Codex acceptance/audit, and Gemini contrast when available. Model-specific waves fail full-run shape unless the user explicitly asks for them.
- The visible header reports only model plus number, for example `Subagents: opus high 12, sonnet high 99, codex 5.5 high 12, gemini 12`. Internal transport labels stay in receipts, not the header.
- Lagging branches do not freeze the run. A stalled model/tool gets isolated behind a parallel debug branch and is marked blocked, deferred, or killed while unrelated wave work continues.
- Answer body is normal chat text, not a Markdown report. Compositions are Follow-up-only. Default Follow-up uses lean lane/composition next-input options with `🪄 Follow-up`, `Pre-run status/score`, and `Audit` fields. Voice rerun menus appear only when the user explicitly asks for them, and Factory/Strategy/Systems stay in the voice set and inform context follow-ups.
- Validation report has zero findings before packaging.

- Wiki mirror is complete.

## Fail Conditions

Reject the packet if:

- v2.8 becomes a renamed v2.7.
- full MMMs collapse to compact-scale line counts.
- follow-ups become labels without next-input, pre-run, and audit language.
- compositions appear as body catalogs instead of follow-up prompts.
- external model advice is copied without Codex acceptance.
- runtime-specific adapters become the universal wiki source.
- behavior claims are made without behavior receipts.
