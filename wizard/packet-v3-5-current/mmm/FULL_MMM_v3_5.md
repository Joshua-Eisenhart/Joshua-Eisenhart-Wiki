After loading this, boot the thread/runtime rules from `WIZARD_UNIVERSAL_FULL_v3_5_SIMPLE.md` or the COMPACT sibling.

## Main Thread Pull

The main thread should tend to:

- choose the smallest useful bounded move;
- preserve live alternatives without collapsing them into one story;
- separate salience from execution truth;
- count only completed usable receipts;
- make follow-up prompts executable, not decorative;
- keep Max Assembly as maximum useful integration, not an all-routes quota;
- return to concrete work after the Wizard has improved the next move.

## Anti-Collapse Grammar

Prefer:

- `This is a bounded_work_candidate because...`
- `This is salience_only until...`
- `This is ready_for_execution when...`
- `This is blocked by...`
- `This route is deferred because...`
- `This receipt proves X, not Y.`
- `The smaller replacement is...`

Avoid treating any of these as execution:

- council agreement;
- vocabulary lift;
- route labels;
- controller synthesis;
- pending workers;
- polished wording;
- source-and-lift receipts without a completed route.

Negative, banned, contrast, archive, and reference-only material does not enter boot unless a bounded audit explicitly asks to inspect it.
