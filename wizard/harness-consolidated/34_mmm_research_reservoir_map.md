# MMM Research Reservoir Map

Purpose: route external and internal sources into the MMM word ratchet without turning them into imitation targets or unexamined authority.

Status: first-pass research map for building deeper wiki language samples. This page is a source-routing surface, not a bibliography closure. For the current sentence/noun-level MMM upgrade, see [[mmm-formal-noun-and-great-sentence-reservoir]].

## Core rule

External writers and research fields are reservoirs, not authorities over the system.

The authority source is the wiki/harness and the root constraints. External material is admitted only where it helps build better system language under:
- `a = a iff a ~ b under M`
- finitude
- non-commutation
- non-Cartesian framing
- anti-Platonic restraint
- constraint on distinguishability
- correlation / symmetry / probability before object-language
- earned hierarchy rather than primitive rank

### Technique reservoir

Use these to study how dense language shapes thought, not to imitate their surface voice:
- Hobbes: definition-density, repeated terms, political ontology built by language
- Shakespeare: voice under pressure, role, ambiguity, reversal, cadence, conflict
- Hume / Montaigne / James: plain sense, restraint, experience, habit, sentiment, no overreach
- Confucian classics: role, ritual, cultivated hierarchy, rectification of names

- legal reasoning: burden, evidence, admissibility, witness, exhibit, chain of custody
- craft manuals / field notes: apprenticeship, observed skill, situated practice, not rule-only knowing

Do not make the system sound like these writers. Extract word fields, grammar moves, and pressure-holding techniques; rewrite them in Codex Ratchet language.

## Research lanes to add or deepen

### 1. Hidden Primitive Identity Audit

Question:
Where does a field require stable identity before it has earned distinguishability conditions?

Audit targets:
- equality in logic
- extensionality in set theory
- identity morphisms in category theory
- identity types in type theory
- definitional equality in proof assistants
- object permanence in physics
- variable identity in formal models
- process identity in computer systems
- person / role / institution continuity

Aligned language:
- no probe distinguished
- same at this resolution
- equal only as shorthand
- identity admitted under M
- object-language deferred
- name carry not yet earned

Caution:
Category theory is useful because it weakens object-essentialism, but it is not automatically clean. Its identity morphisms, equality of arrows, object persistence, and commuting diagrams need audit.

### 2. Operational Equivalence / Measurement

Question:
What does it mean for a term to have meaning only through operations, apparatus, and returned surfaces?

Source families:
- operationalism
- measurement theory
- observables in physics
- calibration and control theory
- model-based measurement

Vocabulary:
operation, apparatus, observable, readout, calibration, procedure, instrument, signal, trace, returned value, measurement surface, control, baseline, ablation

Sentence stems:
- The term earns meaning only where the operation returns a distinction.
- The apparatus did not expose that difference.
- The claim is stronger than the measurement surface.

### 3. Constraint Engineering / Formal Methods

Question:
How do constraints exclude candidates without turning the survivor into a metaphysical winner?

Source families:
- finite-domain CSP
- SAT / SMT / MaxSAT
- answer set programming
- constraint propagation
- model checking
- TLA+ / Alloy
- Hoare logic / separation logic
- Lean / Coq / Isabelle
- abstract interpretation
- runtime verification

Vocabulary:
constraint, satisfiable, unsatisfiable, model, countermodel, witness, certificate, invariant, obligation, proof surface, proof kernel, unsat core, finite model, refinement, counterexample, trace

Caution:
A machine-checked proof proves a statement inside a formal system. It does not by itself prove that the primitive ontology of that formal system is clean.

### 4. Distinguishability / Contextuality / Quantum Identity

Question:
How do measurement context and indistinguishability determine what can be named as state, value, or object?

Source families:
- Kochen-Specker contextuality
- operational quantum logic
- quantum resource theories
- indistinguishable particles
- quasi-set theory and non-individuals
- density matrices and observable algebras

Vocabulary:
context, observable, basis, projection, operator, commutator, noncommuting, measurement context, indistinguishable, operational equivalence, state preparation, effect, POVM, density matrix, trace, tensor, entanglement, symmetry

Caution:
Quantum language can drift mystical or object-like. Keep it tied to measurement contexts, operators, and distinguishability surfaces.

### 5. Non-commutation / Concurrency / Trace Theory

Question:
Where is order part of the object rather than a detail of execution?

Source families:
- noncommutative algebra
- concurrency theory
- distributed systems
- trace theory
- partial orders
- event structures
- memory consistency
- race detection

Vocabulary:
order, sequence, trace, interleaving, partial order, dependency, happens-before, lock, race, consistency, commutator, noncommuting, swap, reversal, path, schedule, queue, handoff

Sentence stems:
- A then B is not B then A unless the reversed probe earns equivalence.
- The trace is part of the claim.
- The schedule changed the surface, so the receipt must name it.

### 6. Correlation / Symmetry / Probability Before Object

Question:
How do correlation, symmetry, and probability function as earlier surfaces than object-language?

Source families:
- probability theory, but audited for primitive sample-space assumptions
- statistical mechanics
- stochastic thermodynamics
- information theory
- symmetry / group actions
- invariants and equivalence classes
- relational / structural physics, with anti-Platonic caution

Vocabulary:
correlation, co-variation, distribution, frequency, likelihood, ensemble, invariant, orbit, symmetry, quotient, equivalence class, entropy, channel, signal, noise, compression, redundancy, fluctuation

Caution:
Structural realism is useful but can become Platonic if structure is treated as a self-standing abstract object.

### 7. Attractor / Viability / Dynamical Systems

Question:
How does a candidate remain within an admissible region under perturbation and time-ordering?

Source families:
- attractor basins
- viability theory
- dynamical systems
- control theory
- cybernetics
- affordance theory

Vocabulary:
attractor, basin, trajectory, phase space, state space, gradient, separatrix, perturbation, hysteresis, bifurcation, metastability, relaxation, feedback, regulation, homeostasis, affordance, viable set, control set

Sentence stems:
- The completion fell into the old basin.
- The reservoir changes the low-energy path.
- The candidate stayed inside the viable set under the perturbation tested.

### 8. Evolution / Selection / Ratchet

Question:
How do candidates vary, get selected, persist, mutate, and leave lineage without becoming necessary outcomes?

Source families:
- Darwinian selection
- evolutionary epistemology
- cultural evolution
- population thinking
- replicator dynamics
- niche construction
- exaptation

Vocabulary:
variation, selection, retention, lineage, inheritance, mutation, recombination, drift, fitness surface, pressure, niche, bottleneck, founder effect, population, generation, descent, pruning, extinction, adaptation, exaptation

Sentence stems:
- The survivor is not the destined form. It is the form not excluded under this pressure.
- The lineage carries receipts, not essence.
- Drift is real where selection did not constrain the variation.

### 9. Language / Semiotics / Ordinary Usage

Question:
How do naming, signs, usage, grammar, and ordinary words shape the next thought?

Source families:
- Peircean semiotics
- Saussure and structural linguistics, with caution
- ordinary-language philosophy
- pragmatics
- corpus linguistics
- rhetoric and poetics
- translation studies
- Confucian rectification of names
- Míng Jiā / School of Names

Vocabulary:
name, sign, token, mark, inscription, usage, register, phraselet, clause, cadence, grammar, stem, reference, contrast, rectification, role-title, conduct, designation, translation, gloss

Sentence stems:
- The title is admitted only where conduct still couples to it.
- The word is not a free-standing object; it is a constraint on future use.
- The phraselet enters the reservoir only if it changes continuation pressure.

### 10. Ritual / Role / Hierarchy / Institution

Question:
How does order persist through repeated practice, asymmetry, role, lineage, office, and correction?

Source families:
- Confucian ritual and role ethics
- anthropology of ritual
- institutional theory
- apprenticeship and craft transmission
- legal offices and procedural authority
- military / monastic / guild order, with caution

Vocabulary:
ritual, rite, role, office, hierarchy, rank, deference, obedience, duty, obligation, lineage, inheritance, precedent, practice, conduct, reverence, remonstrance, correction, apprenticeship, transmission

Aligned distinction:
- admitted order: earned, observed, inherited-with-receipts, role-borne, practice-maintained
- excluded order: title as truth, rank without conduct, obedience as epistemic surrender, authority immune to correction

Pushback to preserve:
Flattening hierarchy is not the system. The system rejects primitive equality as much as primitive rank.

### 11. Recursive Context / Wiki as Environment

Question:
How should the wiki be used as an external language environment rather than compressed into one prompt?

Source:
- Alex L. Zhang, Tim Kraska, Omar Khattab, `Recursive Language Models`, arXiv:2512.24601.

Paper claim surface:
RLMs treat long prompts as part of an external environment and let the model inspect, decompose, and recursively call itself over snippets. The abstract reports inputs up to two orders of magnitude beyond model context windows and stronger performance than vanilla long-context scaffolds across several long-context tasks.

System use:
- wiki / harness = external language environment
- controller = inspector and task router
- mini-MMM = local saliency slice
- subagent = recursive call over bounded source slice
- receipt = local witness
- synthesis = parent reconstruction with anti-collapse audit

Caution:
This supports recursive context handling. It does not by itself prove voice mini-MMM saliency or council independence.

## Writer-technique non-imitation rule

Admitted extraction from great writers:
- repetition density
- definition chains
- pressure scenes
- role-maintaining grammar
- contrastive phrasing
- ordinary-word force
- cadence families
- sentence motion under tension

Excluded extraction:
- archaic surface style
- signature metaphors
- proper-name aura
- long source-adjacent phrases
- genre costume
- `make Hume sound old`, `make Strategy sound like Sun Tzu`, `make Systems sound cybernetic`

The target is not Shakespeare, Hobbes, Hume, Confucius, or Zhuangzi. The target is Codex Ratchet language at comparable density.

## Research workflow

1. Start from the internal language fingerprint.
2. Choose one external source family.
3. Extract common words, technical words, phraselets, and sentence motion.
4. Mark each candidate admitted / context-limited / quarantined / excluded.
5. Write Codex Ratchet sample sentences in the system's own grammar.
6. Add only the survivors to the word ratchet.
7. Test a mini-MMM against null-vs-loaded workers.
8. Promote only after the language changes output, not after it looks impressive.
