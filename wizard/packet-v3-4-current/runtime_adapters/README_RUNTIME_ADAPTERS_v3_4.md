# Runtime Adapters v3.4

Runtime adapters translate the Universal Core into local execution behavior. They are not higher canon. If an adapter conflicts with `universal_core/WIZARD_UNIVERSAL_CORE_FULL_v3_4.md`, the Universal Core wins.

Adapters are allowed in v3.4 because this packet is deployment-oriented. They do not boot as the main law unless the local runtime explicitly selects one after loading the positive main MMM and Universal Core.

## Adapter contract

Each adapter must preserve:

- positive main MMM boot before task;
- main-agent / subagent MMM separation;
- spawned / blocked / deferred / simulated route truth;
- controller synthesis is not execution;
- Audit fixes the answer rather than becoming a default section;
- Follow-up Make -> Run/Scout -> Audit/Improve;
- elastic wave sizing rather than hard worker counts.

## Available adapters

- `CODEX_RUNTIME_ADAPTER_v3_4.md` — Codex as repo/file/tool/terminal executor and validator.
- `CLAUDE_RUNTIME_ADAPTER_v3_4.md` — Claude as long-context semantic, voice, and audit worker.
- `HYBRID_CODEX_CLAUDE_RUNTIME_ADAPTER_v3_4.md` — Codex controller/executor with Claude subagents.
