# Wizard QIT Build Mode v2.7

Wizard build mode uses the full Wizard system to move QIT engines and sim plans toward runnable evidence. It is not an audit-only mode.

## Read Pack

Load the universal wiki reference first:

1. `wiki/harness/SALIENCE_LOADER.md`
2. `wiki/harness/00_READ_FIRST.md`
3. `wiki/wizard/00-read-first.md`
4. `wiki/wizard/01-wizard-general.md`
5. `wiki/wizard/02-mmm-reference.md`
6. `wiki/wizard/06-qit-build-read-pack.md`
7. `wiki/wizard/07-sim-admission-router.md`
8. `wiki/wizard/08-proof-graph-gates.md`

Then read the active project and sim plan files named in the wiki read pack.

## Build Contract

Every QIT build-mode run must produce one of:

- a sim rerun or repair target
- a proof/graph gate result
- a wiki reference improvement
- a packet/MMM adapter improvement
- a status-label correction
- a bounded follow-up composition

## Required Full Wizard Routes

Use all voices, all lanes, checks/guards, system routes, and compositions when changing the Wizard/MMMs or making a QIT engine planning decision. If a route did not run, mark it blocked or deferred rather than hiding it.

## Claim Ceiling

Proof, graph, and sim evidence promotes only the claim it actually tested. The Wizard must name what stayed open.
