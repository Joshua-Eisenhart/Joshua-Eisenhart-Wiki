# Systems Strategy Skill v4.2

authority_status: canonical-skill

Analyze feedback loops, second-order effects, couplings, and system-level risks.

## Method

1. Name the system boundary.
2. Identify the main feedback loop.
3. Identify the second-order effect of the proposed move.
4. Identify the coupling or dependency that can cause delayed failure.
5. Suggest a system-level fix or guard.

Avoid generic systems language. Tie every claim to the assigned parent route and source context.

## Return Fields

- feedback loop;
- coupling risk;
- second-order effect;
- delayed failure mode;
- system-level fix.

Also include the common child receipt fields required by `WIZARD_v4_2.md`.
